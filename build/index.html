    Welcome to More Recipes API

    Welcome to the More Recipe API Documentation! You can use our API to access More Recipes API endpoints, which can get information on various Recipes in our database. Users can perform various actions like Sign Up, Sign In, Add Recipe, Edit Recipe, Delete Recipe, and other features.

    Authentication

    More recipes platform needs users to be authenticated to be able to perform the actions specified above. On a successful sign up or sign in, a token is generated which gives users access to this privileges.

    Sign Up

    Request

    • Sign up end point: POST /api/v1/users/signup
    • Body (application/json)

    Response

    • Status: 201 Created
    • Message: Registration successful
    • Body (application/json)

    This end point is used to register/create a new user

    HTTP Request

    POST https://hosted-url.heroukuapp.com/api/v1/users/signup

  "lastName": "Mohammed",
  "username": "mohzak",
  "email": "example@gmail.com",
  "password": "arandomstring",
  "confirmPassword": "arandomstring"
}

    Response:

    {
  "message": "Registration successful",
  "user": {
    "id": 1,
    "username": "mohzak"
  },

    Errors

    The More Recipes API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request sucks.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The recipe requested requires access.
    404 Not Found -- The specified recipe could not be found.
    410 Gone -- The recipe requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
